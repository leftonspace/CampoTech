# CampoTech High-Level Architecture

**Version:** 1.0
**Last Updated:** December 2024
**Phase:** 9.11 Technical Architecture Documentation

## System Overview

CampoTech is a field service management platform built for Argentine SMBs. The architecture prioritizes:
- **Mobile-first** design for smartphone-only users
- **WhatsApp-first** communication for the Argentine market
- **Offline-capable** operations for field workers
- **Multi-tenant** isolation with organization-scoped data

## Component Responsibilities

### Client Tier

| Component | Technology | Responsibility |
|-----------|------------|----------------|
| Mobile App | React Native (Expo) | Primary interface for field workers |
| Web Dashboard | Next.js 14 | Back-office management |
| WhatsApp | Business API | Customer communication |
| External Webhooks | HTTP | Third-party integrations |

### API Gateway

| Component | Responsibility |
|-----------|----------------|
| Next.js API Routes | Request routing, response handling |
| Authentication Middleware | JWT validation, session management |
| Rate Limiting | Per-user, per-org limits via Redis |
| Request Validation | Zod schema validation |
| Error Handling | Standardized error responses |

### Service Tier

#### Domain Modules (`src/modules/`)

| Module | Key Files | Responsibility |
|--------|-----------|----------------|
| Users | `index.ts`, `onboarding/` | User CRUD, verification, onboarding |
| Jobs | `index.ts`, `state-machine.ts` | Job lifecycle, state transitions |
| Customers | `index.ts` | Customer database, search |
| Invoices | `index.ts` | AFIP electronic invoicing |
| Tracking | `tracking.service.ts`, `eta/` | Live location, ETA calculation |
| Notifications | `notification.service.ts` | Multi-channel delivery |
| Payments | `index.ts` | Payment recording, balance |
| Audit | `audit.service.ts` | Immutable audit logging |

#### Integrations (`src/integrations/`)

| Integration | Key Components | External Service |
|-------------|----------------|------------------|
| WhatsApp | Messages, Aggregator, Templates, Webhook | Meta Business API |
| MercadoPago | OAuth, Preferences, Webhook | MercadoPago API |
| AFIP | WSAA, WSFEv1, Padron | AFIP Web Services |
| Voice AI | Transcription, Extraction | OpenAI Whisper/GPT |
| Maps | Providers, Routing | Google Maps/Mapbox/OSM |

### Data Tier

| Store | Technology | Data Types |
|-------|------------|------------|
| PostgreSQL | Prisma ORM | Relational data, transactions |
| Redis | Upstash | Sessions, queues, cache |
| Object Store | S3/R2 | Binary files, documents |

## Cross-Cutting Concerns

### Security
- Authentication via OTP (WhatsApp/SMS)
- Authorization via RBAC (5 roles)
- Row-Level Security on all tables
- Encryption at rest (AES-256-GCM)
- Webhook signature validation

### Observability
- Structured logging (JSON)
- Request tracing
- Audit trail with hash chain
- Performance metrics

### Resilience
- Circuit breakers for external services
- Queue-based retry with backoff
- Graceful degradation
- Panic mode for critical failures

### Multi-Tenancy
- Organization-scoped data
- Tenant-aware job queues
- Isolated file storage
- Per-org rate limits
